.1 ■ 概要

  ペン字風の手書きTrueType/OpenType和文フォントです。

  Aシリーズは、ひらがなカタカナがアンチック体風になっています。それ以外
  の部分については上記のペン字風と同等です。

  Kシリーズはちょっとくずした毛筆風、Mシリーズは全くくずしていない端正な
  毛筆風の、手書きTrueType和文フォントです。

..1-1 ◆ TrueType版

...1-1-1 ▼ 標準版

      英数かな記号 (縦書き専用文字を含む)、JIS第1水準漢字、JIS第2水準漢
      字、NEC/IBM選定拡張文字に加えて、Unicode (「CJK Unified 
      Ideographs Extension A (U+3400～U+4DBF)」「CJK Compatibility 
      Ideographs (U+F900～U+FA6A)」「CJK Unified Ideographs Extension B 
      (U+20000～)」に存在するJIS第3～4水準の総ての漢字が収録されていま
      す。

      その他、JISX0213:2004で追加された漢字や、OpenType規格 (Adobe 
      Japan 1-4, 1-5) で採用されているUnicodeが割り付けられているがJIS第
      1～4水準やIBM漢字に入っていない漢字 (ほとんどが補助漢字) などが
      入っています。

      Unicode BMPに含まれないJIS第3～4水準漢字 (CJK Unified Ideographs 
      Extension Bに入っている漢字) 302文字 (303文字!?) は、BMPの以下の場
      所にも収録しています。

      1. Undefined Area 「U+AAA1～U+ABCF」(Yi Radicals: U+A490～U+A4CFと
        Hangul Syllables: U+AC00～U+D7AFの間の空白領域)。このコード割り
        当ては、JISがUnicode Consortiumに提案した括弧付きコードを採用し
        ています。JIS規格「JISX0213/7ビット及び8ビットの2バイト情報交換
        用符号化拡張漢字集合」の「JIS X 0221からの索引」(P456～) の内、
        P503～P505に掲げられているものです。

      2. Private Area 「U+F711～U+F8EA」。このコード割り当ては、
        「charset.info」(http://charset.info/) の「Shift_JIS-2004 (JIS 
        X 0213:2004) vs Unicode mapping table」
        (http://charset.info/sjis-2004-std.txt) に記載されている代替用
        「UCS private area mapping」に従っています。

....1-1-1-1 □ 外字について

        総てのフォント (毛筆系の一部を除く) には、外字が内蔵されていま
        す。通常の外字として内蔵していますので、Unicodeの扱えない (シフ
        トJISだけしか扱えない) アプリからでも使用することができます。

        ・ F040～F4FC　囲み英数字など
        ・ F540～F5E8　JIS X 0213:2004で例示字形の変更された漢字の、
           　　　　　　JIS83(90)字形
        ・ F5E9～F5F2　JIX X 0213:2004で第3水準に追加された10文字
        ・ F7F3～F5FC　(今のところ空き)
        ・ F640～F6FC　組数字、記号、濁点付きかな、組記号など
        ・ F740～F78F　和文用カッコ類
        ・ F790～F79F　横書き用と縦書き用の引用符‘’“”〝〟
        ・ F7A0～F7B0　(今のところ空き)
        ・ F7B1～F7C4　第3水準にしかない新人名用漢字
        ・ F7C5～F7CB　K-JIS、大日本漢和辞典より
        ・ F7CC～F7F6　拡張漢字
        ・ F7F7～F7FB　(今のところ空き)
        ・ F7FC　　　　ユーロ記号
        ・ F840～F9F1　拡張漢字
        ・ F9F2～F9FC　括弧付き漢数字

....1-1-1-2 □ JIS X 0213:2000対応状況

        JIS X 0213 (所謂JIS2000。JIS第3～4水準) でCJK Unified 
        Ideographs Extension Bに入っている302文字も、そのままの文字コー
        ドで収録できるようになりました。これは、charset.info 
        (http://charset.info/) で配布されている「unicmap/505氏作」を用い
        て、JIS2000に含まれる文字総てをUnicode上に配置できるようにした
        フォントファイルを用いて作られたものです。

        当然のことではありますが、CJK Unified Ideographs Extension Bにア
        クセスできるアプリが必要になります。Microsoft OfficeのWord/Excel
        なら、「挿入 (I)|記号と特殊文字 (S)」で「フォント (F)」に
        Unified Ideographs Extension Bを含むフォントを指定すると、選択で
        きます。

        ただし、現在、総てのアプリでExtension Bのエリアにアクセスできる
        訳ではありません。そのため、Extension Bにある302文字を、前述の通
        り「Undefined Area (U+AAA1～U+ABCF)」と「Private Area (U+F711～
        U+F8EA)」にも収録しています。

....1-1-1-3 □ JIS X 0213:2004対応版

        2004/2/20に出されたJIS X 0213:2004に対応させたもので、168文字の
        例示字形の変更を行いました。ただし、明朝体を基準にした例示字形の
        変更ですので、手書きで行う場合には全く差のないと思われる字もいく
        つかありますので、そういった文字については変更していません。

....1-1-1-4 □ SJIS版

        内部がUnicodeでないOS (Windows 95、Windows 98、Windows Me) で
        は、Shift JISコードの割り振られている文字しか扱うことは出来ませ
        ん。そのため、フォントファイルにちゃんと内蔵しているのに使えない
        文字が多数存在しています。

        入っているのに使えない文字のためにフォントファイルのサイズが大き
        くなるのは無駄です。そこで、Shift JISでアクセスできる範囲のみの
        漢字に絞ったフォントファイルを作成しました。これを、SJIS版と呼び
        ます。

        SJIS版は、通常のフォントと比べて、内蔵している文字の数(および
        ファイルサイズ)が異なるだけです。フォント名なども全く同じなの
        で、間違えないようにお気を付け下さい。

        なお、内部がUnicodeでないOS (Windows 95、Windows 98、Windows 
        Me) で通常版を使うと、一部、縦書き用グリフがおかしくなると言う不
        具合が報告されています。そのような場合には、このSJIS版を使うこと
        で正常になるようです。

...1-1-2 ▼ あんちっく仮名版 -- Aシリーズ

      Aシリーズは、標準のペン字風を、ひらがなカタカナのみ、アンチック体
      風にしたものです。それ以外は標準と同等です。

...1-1-3 ▼ 新かな版 -- Nシリーズ

      Nシリーズは、標準のペン字風を、ひらがなカタカナのみ、より懐の広い
      モダンな感覚の字にしたものです。それ以外は標準と同等です。

...1-1-4 ▼ ちょっとくずした毛筆版 -- Kシリーズ

      Kシリーズは、英数かな記号、第1、2水準、NEC/IBM漢字まで完成していま
      す。

...1-1-5 ▼ あんちっく仮名版 -- KAシリーズ

      なお、Kシリーズのひらがなカタカナのみ、アンチック体風にした
      Y.OzFontKAもあります。

...1-1-6 ▼ くずしていない毛筆版 -- Mシリーズ

      Mシリーズは、英数かな記号、第1水準漢字まで完成しています。第2水
      準、NEC/IBM漢字は、随時追加中ですが、既に、使用頻度の高そうな文字
      は収録済みです。

..1-2 ◆ OpenType版

    基本的には、TrueType版の文字の内、Adobe Japan 1-4規格にUnicode付きで
    登録されているもののみが使用できます。

    なお、Unicodeの割付のないグリフであっても、Y.OzFont TrueType版の外字
    に収録されている「囲み英数字」と「漢字」、所謂半角英数字の等幅版文字
    などについては、AJ 1-4規格通りの場所に収録されています。こういったグ
    リフを使用するには、InDesignなどのOpenTypeに対応したアプリが必要で
    す。

    今後、予告無く改良/修正が行われることがあります。

....2-1-1-1 □ フォント単体ではない状態での再配布

        にも入れて頂いていたりしますが、このCDおよびCDイメージを再配布す

        ように他のアプリの一部として再配布する(自動的に再配布してしまう)

...2-1-2 ▼ 改変および改変物の再配布

      手書きフォントという性格上、新たに字を作成するのは難しいかも知れま
      せんが、改変および改変したものの再配布もかまいません。この場合の改
      変とは、フォントのフォーマット変換や、文字そのものの改造・追加作成
      などを言います。

      ただし、以下の条件に合う形でお願いいたします。

....2-1-2-1 □ 改変したものを個人的にご使用になられる場合

        ・ 印刷しての使用は、自由にできます。

        ・ 義務はありませんが、メールにてご連絡いただけると嬉しく思いま
          す。

....2-1-2-2 □ 改変したものを再配布される場合

        ・ 元になるフォント名とそのバージョン、著作権および配布元を明記

          http://yozvox.web.infoseek.co.jp/

        ・ メールにてご連絡ください。

.4 ■ 使用方法

..4-1 ◆ Windowsの場合

...4-1-1 ▼ フォントのインストール

      同梱のSetup.exe (フォントインストーラ、安沢氏作) を実行してくださ
      い。
      ・ 武蔵システム　http://musashi.or.tv/

      手動でインストールする場合は、ダウンロード&解凍したファイルを、
      「コントロールパネル|フォント|ファイル|新しいフォントのインストー
      ル」でインストールしてください。既に旧版が入れてある場合には、先に
      削除してからコピーしてください。

...4-1-2 ▼ フォントのアンチエイリアス(スムージング)

      TrueTypeフォントを使う場合、「コントロールパネル/画面/デザイン/効
      果/□次の方法でスクリーンフォントの縁を滑らかにする/標準⇔
      ClearType」(Windows XPの場合)を用いて、画面上のフォントにアンチエ
      イリアスを掛けることができます。これにより、Y.OzFontなどのビット
      マップ内臓ではないフォントでも、画面上滑らかで見やすい表示にするこ
      とが可能です。

      しかし、18ポイント以下の小さいサイズで使う場合、アンチエイリアスを
      掛けるとかえって見難くなるだろうからと言う配慮からか、掛かりませ
      ん。そのような場合にもアンチエイリアス処理を行わせるには、本サイト
      でダウンロードできる「Font Smoother」をお使いください。フォント
      ファイルを書き換えて、小さいサイズでもアンチエイリアスされるように
      なります。

      ※ 現在配布のフォントは、既にFont Smoother処理を済ませたものになっ
        ています。
